Política de Privacidad

Última actualización: abril 2025 · BarOrder v3.0

BarOrder es una aplicación para gestionar pedidos colaborativos en bares y restaurantes. Nos tomamos muy en serio tu privacidad. Esta política explica qué datos recogemos, para qué y cómo los protegemos.

1. Datos que recopilamos

  • Nombre de usuario: el nombre que escribes al unirte a un grupo (apodo, nombre real o cualquier nombre). No requerimos email ni contraseña.
  • Nombre del grupo: el nombre que el administrador da al grupo.
  • Pedidos realizados: las bebidas y aperitivos que seleccionas dentro de la sesión.
  • Mensajes de chat: los mensajes enviados en el chat interno del grupo.

No recogemos: dirección de correo, contraseñas, número de teléfono, ubicación, datos de pago, historial de navegación ni ningún identificador personal.

2. Cómo usamos los datos

  • Mostrar los pedidos en tiempo real a todos los miembros del grupo.
  • Calcular totales y rondas de pago dentro de la sesión.
  • Permitir el chat interno del grupo durante la sesión.

Los datos no se usan para publicidad, no se venden a terceros y no se emplean para ningún análisis externo.

3. Almacenamiento y retención

Los datos de cada sesión (grupo, miembros, pedidos, chat) se almacenan en Supabase (proveedor de base de datos en la nube, con servidores en la UE). Los datos se eliminan automáticamente cuando el administrador cierra el grupo. El tiempo de vida máximo de una sesión activa sin cerrarse es de 24 horas.

En tu dispositivo, la aplicación guarda localmente (AsyncStorage) únicamente el identificador de sesión y las preferencias del catálogo. Estos datos también se eliminan al cerrar el grupo o al cerrar sesión.

4. Terceros

  • Supabase (supabase.com) — base de datos y tiempo real. Política de privacidad: supabase.com/privacy
  • Vercel (vercel.com) — alojamiento de la web companion. Política de privacidad: vercel.com/legal/privacy-policy
  • Expo / React Native — framework de la aplicación móvil. No envía datos propios del usuario a Expo.

5. Cámara

La aplicación puede solicitar acceso a la cámara únicamente para escanear códigos QR de precios generados por el administrador del bar. La cámara se activa solo cuando el usuario abre manualmente el escáner y no graba ni transmite vídeo en ningún momento.

6. Derechos del usuario

Al ser una aplicación de sesiones temporales sin registro, no almacenamos datos vinculados a una identidad permanente. Si deseas que se eliminen los datos de una sesión antes de que expire, el administrador del grupo puede cerrar el grupo manualmente, lo que borra todos los datos inmediatamente.

7. Seguridad

Las comunicaciones entre la app y Supabase se realizan mediante HTTPS. Las claves de acceso a la base de datos están limitadas a operaciones de lectura/escritura dentro del mismo grupo (Row Level Security de Supabase).

8. Contacto

Si tienes preguntas sobre esta política de privacidad, puedes contactarnos en: barorderapp@gmail.com

Privacy Policy (English)

Last updated: April 2025 · BarOrder v3.0

BarOrder is an app for collaborative drink ordering at bars and restaurants. We take your privacy seriously. This policy explains what data we collect, why, and how we protect it.

1. Data We Collect

  • Username: the name you enter when joining a group (nickname, real name or any name). No email or password required.
  • Group name: the name the administrator gives to the group.
  • Orders placed: the drinks and snacks you select during the session.
  • Chat messages: messages sent in the group's internal chat.

We do NOT collect: email address, passwords, phone number, location, payment data, browsing history or any persistent personal identifier.

2. How We Use Data

  • Display orders in real-time to all group members.
  • Calculate totals and round payment rotation within the session.
  • Enable in-session group chat.

Data is never used for advertising, never sold to third parties, and never used for external analytics.

3. Storage & Retention

Session data (group, members, orders, chat) is stored in Supabase (cloud database provider with EU servers). Data is deleted automatically when the administrator closes the group. Maximum session lifetime without closing is 24 hours.

On your device, the app stores locally (AsyncStorage) only the session identifier and catalog preferences. These are also deleted when the group is closed or the user logs out.

4. Third Parties

  • Supabase (supabase.com) — database and real-time. Privacy policy: supabase.com/privacy
  • Vercel (vercel.com) — web companion hosting. Privacy policy: vercel.com/legal/privacy-policy
  • Expo / React Native — mobile app framework. No user data is sent to Expo.

5. Camera

The app may request camera access solely to scan QR price codes generated by the bar administrator. The camera is activated only when the user manually opens the scanner and does not record or transmit video at any time.

6. User Rights

Since BarOrder uses temporary sessions without registration, no data is linked to a persistent identity. If you want data from a session deleted before it expires, the group administrator can close the group manually, which immediately deletes all data.

7. Security

Communication between the app and Supabase uses HTTPS. Database access keys are restricted to read/write operations within the same group (Supabase Row Level Security).

8. Contact

If you have questions about this privacy policy, contact us at: barorderapp@gmail.com
